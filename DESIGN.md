### 1.4 图标系统

| 类别 | 规格 | 说明 |
|------|------|------|
| **风格** | SF Symbols 6 (iOS 原生) | 统一使用 Apple 原生图标，保证与 iOS 系统视觉一致 |
| **线宽** | Regular (默认) / Medium (选中态) | Tab 选中态图标加粗至 Medium |
| **尺寸** | 24px (导航) / 20px (内联) / 16px (标签内) | 统一基准网格 |
| **自定义图标** | 衣橱分类图标采用定制 SVG | 上装、下装、外套、鞋履、包袋、饰品六类使用专属图标 |
| **颜色** | 继承父级文字色 | 图标不单独设色，通过 opacity 控制层级 |

### 1.5 间距系统

基于 **4px 基准网格**，所有间距为 4 的倍数：

```
Spacing Scale (4px base)
────────────────────────
  xs:   4px    (紧密关联元素)
  sm:   8px    (图标与文字、标签间距)
  md:   12px   (列表项内间距)
  lg:   16px   (卡片内边距、组件间距)
  xl:   20px   (区块间距)
  2xl:  24px   (页面水平边距、Section 间距)
  3xl:  32px   (大区块分隔)
  4xl:  40px   (页面顶部留白)
  5xl:  48px   (页面间过渡区)
```

**页面安全区：**
- 水平 padding: 20px (iphone 默认 safe area 兼容)
- 顶部: 遵循 SafeAreaView，status bar + 16px
- 底部: 遵循 SafeAreaView，Tab Bar 高度 88px (含 safe area)

### 1.6 圆角系统

```
Radius Scale
────────────────
  none:   0px     (表格、分隔线)
  sm:     6px     (小型 Chip、标签、徽章)
  md:     10px    (按钮、输入框、缩略图)
  lg:     14px    (卡片、面板、图片容器)
  xl:     18px    (大卡片、Modal)
  full:   9999px  (药丸按钮、头像)
```

### 1.7 阴影与层级

```
Elevation Scale (从 Flat UI 到浮层)
────────────────────────────────────

flat (0): 无阴影，纯平面元素
  → 页面背景、分割线

raised (1): 卡片微浮
  shadow: 0px 1px 3px rgba(29,26,27,0.06), 0px 1px 2px rgba(29,26,27,0.04)
  → 列表卡片、衣饰缩略图

overlay (2): 浮层面板
  shadow: 0px 4px 16px rgba(29,26,27,0.08), 0px 0px 0px 0.5px rgba(29,26,27,0.04)
  → 下拉菜单、Tooltip、底部抽屉

modal (3): 模态弹窗
  shadow: 0px 20px 60px rgba(29,26,27,0.15)
  → 弹窗、ActionSheet、全屏预览

emphasis-ring: 聚焦环
  box-shadow: 0px 0px 0px 3px rgba(139,46,77,0.25)
  → 输入框聚焦态
```

### 1.8 动效规范

| 场景 | 时长 | 曲线 | 说明 |
|------|------|------|------|
| 页面转场 (Push) | 300ms | ease-out (cubic-bezier 0.0, 0.0, 0.2, 1.0) | 新页面从右侧滑入 |
| 页面返回 (Pop) | 250ms | ease-in (cubic-bezier 0.4, 0.0, 1.0, 1.0) | 当前页向右滑出 |
| Modal 弹出 | 350ms | spring (stiffness: 400, damping: 40) | 弹性从底部升起 |
| 微交互 (hover/press) | 150ms | ease-out | 按钮缩放 0.97x |
| 列表重排 | 300ms | spring | LayoutAnimation 自动编排 |
| 数字滚动 | 600ms | ease-out | 统计数据递增动画 |

#### 2.2 交互细节

| 交互 | 行为 | 反馈 |
|------|------|------|
| **日期左右滑动** | 水平滑动切换日期，单行条自适应跟随 | 选中日显示 burgundy 圆环背景 (32×32px)，非今日日期 ink-muted 色 |
| **点击"今天"** | 一键回到今日 | 日期条自动滚动至今日位置，spring 动画 |
| **全周视图展开** | 从日期条向下拉拽 | 日期条扩展为 7 天拼图网格 (2行×4列+1)，每个日格内显示当日穿搭缩略图。展开高度 animate 200ms |
| **全周视图收起** | 上滑或点击收起箭头 | 网格收起为单行日期条，恢复单日视图 |
| **穿搭计划卡长按** | 触发 Context Menu | 弹出选项：替换方案、查看详情、移除计划 |
| **移除计划** | 点击移除 | 弹出确认 ActionSheet："移除后对应单品使用频次将-1"，确认后卡片 slide-out 动效 |
| **快速选择穿搭** | 横滑选择已有方案 + 点击 | 方案卡片 fade-in 至今日计划位，关联单品频次自动 +1 |
| **出行任务增删** | 点击 "+" 新增 / 左滑删除 | 新增弹出输入框，用键盘确认；左滑露出红色删除按钮 |

#### 2.3 状态处理

| 状态 | 表现 |
|------|------|
| **空状态 (当日无计划)** | 居中插图 + "今日尚未安排穿搭" + [AI 智能推荐] 按钮 |
| **加载中** | 日期骨架屏 (3 张计划卡 shimmer 动画) |
| **网络错误** | 顶部 Banner: "网络连接异常，显示已缓存数据"，背景 canvas-parchment 变淡 |
| **计划冲突** | 若同一天安排 ≥2 套穿搭，卡片右上角显示橙色圆点 + "多方案" 标签 |
| **历史日期** | 过去日期的穿搭方案卡片略微降低不透明度 (0.7)，只读不可编辑 |

```

        ├── 品牌 (选填, 文本输入 + 品牌自动补全)

    → [保存] → 卡片 fade-in 进入网格首位
```

#### 2.7 筛选与排序交互

| 组件 | 行为 |
|------|------|
| **分类 Chip** | 水平滑动选择，单选。选中后网格 filtered re-render, spring 动画重排 |
| **搜索栏** | 实时模糊匹配 (debounce 300ms)，匹配范围: 品名、品牌、色系。无结果时显示 "未找到匹配单品" 空状态 |
| **排序依据** | 弹出底部 ActionSheet: 按录入时间(默认) / 价格从高到低 / 价格从低到高 / 穿着频次从高到低 / 保暖属性从高到低 |
| **价格滑块** | 双端 Range Slider，实时过滤。范围标签显示 "¥50 - ¥2,400" |
| **冷暖开关** | 三段式 Segmented Control: 轻薄 / 适中 / 厚暖，选中高亮 |

## 四、关键交互流程说明

### 4.1 核心流程一：AI 智能穿搭 (端到端)

```
用户进入 Studio 页
    │
    ▼
在 AI 交互区输入需求文案
(例: "今天要去参加花园婚礼，帮我搭配优雅又不会太正式的造型")
    │
    ▼ 点击 [✨ 生成搭配方案]
    │
    ▼
显示加载态 (3张方案骨架屏 + "AI 正在分析衣橱...")
    │
    ▼ Gemini API 调用 (后端/Edge Function)
    │  - 传入: 用户需求文字 + 衣橱全量数据 (名称/分类/色系/品牌/厚暖度/频次)
    │  - 模型: Gemini 2.5 Pro / Flash
    │  - Prompt 包含: 颜色搭配规则、场合匹配、季节适宜度、频次均衡
    │
    ▼
返回 3 套方案 → 解析并展示
    │
    ├─→ [选用方案A] → 单品装载至画布 → 各单品频次 +1
    │                  → 方案存入 "穿搭套卡"
    │                  → Toast: "已生成穿搭方案"
    │
    ├─→ [查看详情] → Push 详情页 (效果图 + 色卡 + 单品清单)
    │
    ├─→ [全部保存] → 3套方案全部存入套卡
    │
    └─→ [重新生成] → 重新请求 API
```

### 4.2 核心流程二：手动穿搭创作

```
Studio 页 → 点击 [+ 新建穿搭画板]
    │
    ▼
Push 到空白画布 (Canvas Board)
    │
    ▼
从底部抽屉舱浏览衣橱单品
(横滑查看，点击 + 添加到画布)
    │
    ▼
单品进入画布 (默认位置: 画布中央，随机偏移 ±20px)
    │
    ▼
用户自由创作:
  ├── 拖拽: PanResponder 处理 x/y 移动
  │   (惯性滑动 + 边界吸附画布边缘)
  ├── 缩放: PinchGestureHandler, scale 0.5x – 2.0x
  │   (步进 0.1x, 显示当前缩放值标签)
  ├── 旋转: RotationGestureHandler, -180° – 180°
  │   (步进 5°, 松手自动吸附至最近 15° 倍数)
  └── 移除: 长按单品 → 弹出 × 删除按钮
    │
    ▼
可选: 切换画板底色 (底部预设横滑)
可选: 输入灵感描述文字
可选: 绑定妆容配方
    │
    ▼
点击 [保存] → 画布截图生成穿搭预览图
    │
    ▼
方案存入 "我的穿搭画板" 套卡列表
    │
    ▼
Toast: "穿搭方案已保存" + [应用至日程] 快捷操作
```

### 4.3 核心流程三：日程计划闭环

```
方式 A: 从 Studio 套卡装入
  套卡长按 → [应用至日程] → 日期选择器
  → 选择日期 / 默认今天 → 确认
  → 关联单品频次 +1 → Toast 确认

方式 B: 从 Calendar 页快速选择
  Calendar 页 → 快速选择已有穿搭区块
  → 横滑选方案 → 点击
  → 直接装入当天 → Toast 确认

方式 C: AI 生成后直接装入
  AI 方案 → [选用此方案]
  → 弹出日期选择 (今天 / 选择日期)
  → 确认 → 装入日历 + 频次更新

日程操作:
  查看计划 → 点击计划卡 → 展开详情
  替换方案 → 长按 → [替换方案] → 选择新方案
  移除计划 → [移除计划] → 确认弹窗 → 频次 -1
```

### 4.4 核心流程四：拍照录入单品

```
衣橱页 / 美妆页 → 点击 [📷 拍照]
    │
    ▼
ActionSheet: [拍照] / [从相册选择] / [取消]
    │
    ▼ (选择拍照)
调用真机相机 (expo-image-picker / react-native-image-crop-picker)
    │
    ▼
拍照完成 → 预览 + 确认界面
  (支持裁剪 / 旋转 90° / 滤镜: 无)
    │
    ▼ 确认
进入属性录入表单 (详见 2.12 / 2.16)
    │
    ▼ 填写完成 → [保存入库]
    │
    ▼
图片上传至本地存储 / Cloud Storage
    │
    ▼
卡片生成 → 插入网格首位 (spring 动画)
    │
    ▼
Haptic Feedback + Toast: "已录入衣橱"
```

### 4.5 核心流程五：美妆生命周期管理

```
美妆录入时设定:
  - 可用天数 (生命值): e.g. 180 天
  - 开封日期: e.g. 2026-05-20
  - 到期提醒: 开启 (提前30天)
    │
    ▼
系统自动计算:
  剩余天数 = 可用天数 - (今天 - 开封日期)
  健康度 = 剩余天数 / 可用天数
    │
    ▼
状态判定:
  ├── > 90天 → 🟢 健康
  ├── 30-90天 → 🟠 注意
  ├── 0-30天 → 🔴 临期 (推送通知 + 首页横幅)
  └── < 0 → ⚫ 已过期 (归入过期分组)
    │
    ▼
临期预案:
  - Profile 页展示临期列表
  - 美妆页顶部横幅
  - 筛选"临期优先消耗"排序
  - 推荐使用临期品关联的妆容配方

过期处理:
  - 自动归入"已过期"折叠分组
  - 用户可标记 [已用完/已丢弃] → 从列表移除
  - 或 [续用] → 手动延长可用天数
```

## 六、可访问性规范

| 项目 | 标准 |
|------|------|
| 最小触摸区域 | 44×44px (遵循 iOS HIG) |
| 色彩对比度 | 文字与背景 ≥ 4.5:1 (WCAG AA) |
