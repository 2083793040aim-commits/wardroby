Wardroby (智能穿搭美妆助手)
产品需求文档 (PRD)
项目名称	Wardroby (智能穿衣&美妆创意室)
设计主题	Artistic Flair (艺术雅致流派)
核心定位	为追求生活品质的用户量身打造的私享衣橱管家与面部妆容设计室，融合大模型 AI 智能穿衣指导、自由画布拼贴、妆容关联公式及生命力保质期追踪的数字化时尚空间。
简介：
第一个页面是日历。主要展示每天的穿搭计划，左右滑动，上面是日期，下面是穿搭方案，在下面是当日安排。同时这个日历可以展开，就变成按星期排列的视图。
第二个页面是衣柜。你可以通过拍照的方式录入你的衣服，可以设置衣服的属性，比如价格，类别，颜色，季节品牌，尺码等。顶部会有类别框，点击类别框可以筛选，也可以手动输入搜索。同时可以有筛选旋钮，可以按照价格高低，穿着频率，衣服冷暖进行排序或筛选。
第三个页面是中间的主页面，是穿搭加妆容页面。顶部是搜索和穿搭类别框，下方从上到下是你的穿搭方案，第一个穿搭方案的位置有一个加号用于创建新的穿搭方案。这个加号下面有个AI＋的按钮，可以通过Ai自动给你一套穿搭方案。穿搭加号点开是一个画板：底部可以设置背景、单品、照片、妆容，用户可以自行拖动上传的素材进行排版。这些穿搭方案长按可以选择加一和减一控制里面每个单件的使用次数。使用次数也会通过用户安排在日历上后给每个单品增加一次使用次数。
第四个页面是化妆品收纳。顶部是搜索和类别框，同样可以通过拍照录入化妆品，设置品牌，色号，可用天数等信息。类别框里第一个是妆容，需要特殊标记，用于上传用户妆容方案，每个妆容方案，这个妆容方案需要附带上用的所有化妆品信息（眼妆、鼻影、修容、高光、口红、粉底等，上传妆容时用户可以在类别里选择对应的化妆品）。
第五个页面是我的。信息你自行设置，要合理。

2. 核心页面功能需求规格 (Detailed Module Specifications)

2.1 穿搭日程计划 (Calendar Page)
功能描述：管理用户的每日出行日程与专属风格安排。
交互细则：
滑动日期条：支持无限左右滑动。顶部为星期的文字标识，中间为醒目的日期数字。
一键变身全周视图：点击右上角“展开为全周视图”，单行排版柔和收起，变成纵向 7 天的精美九宫格卡片，右上角标明搭配确认状态（绿标 ✓）。
日程关联频次联动（核心规则）：
当用户在某一天关联了一套经典穿搭，全套穿搭卡的“累积穿搭次数”自动递增 +1。
与之相关联的全部成衣单品，其“使用次数”同样自动递增 +1，实现数据的穿透统计。
用户取消本日计划时，次数相应连带 -1。
当日安排：支持手动增加和删除当日的各种琐碎会客、下午茶或差旅活动，通过优雅的橙底暖色标出，防止日常冲突。

2.2 智能衣柜 (Wardrobe Page)
功能描述：拍照录入衣物，管理属性，支持筛选、搜索、排序。
录入流程：
点击“拍照录入”，拉起照相机（支持 3 秒定时自拍，并提供高级法式定制模特图作为默认生成素材以提供完美测试闭环，亦支持用户直接上传本地相册）。
类别筛选标签栏：上衣、裤子、裙子、外套、鞋子、包包、配饰…可自定义添加或修改类别，点击标签 = 筛选对应类别
高级筛选旋钮 / 面板：
（1）价格排序：从低到高 / 从高到低
（2）穿着频率：多→少 / 少→多
（3）衣服色系：冷色 / 暖色 / 中性色 冷暖度（Warmth Level）：设定 1 (清凉/极薄) 到 10 (温暖/极厚) 的冷暖数值，用颜色进度条直观显示。
衣物网格列表：卡片展示：衣物图片 + 名称+使用次数+；点击卡片 = 查看详情 / 编辑

2.3 穿搭创意室 (Studio Page)
功能描述：提供开放的交互画板，供用户无拘无束地拼贴灵感；并通过大语言模型实现穿衣顾问功能。
AI+ 一键智能穿搭 (AI Curator)：
后台调用规则：将当前库内所有衣服的品牌、分类、尺码、冷暖、色彩信息进行结构化清洗，作为 prompt 上下文打包。
大模型推荐机制：通过大模型，根据输入的诉求（如：“今天下雨很清冷，想去画廊散步”）在用户衣柜中挑选最佳搭配。
输出反馈：AI 自动生成包含“穿搭名称”、“美学推荐理由”、“妆容主题建议”及“精选单品列表”的 JSON 结构，并将该方案的核心单品在画板中以堆叠纸壳效果呈现。
手写作画画布 (Interactive Canvas Board)：
高维度交互：
从底部备选舱拖入或点击单品、美妆，物品会化为画作元素出现在背景中。
选中微调：双击或点击画布上的成衣，调出“物料尺寸缩放拖动条 (0.5x - 2.0x)”与“旋转斜度轮盘 (180°/-180°)”，也可以通过上/下/左/右虚拟按键控制平移排版。
底置色彩：一键切换“雨林迷雾（淡绿黄）”、“摩卡格纹（波点网格）”等法式轻熟艺术底色。

2.4 美妆收纳 (CosmeticsPage)
功能描述：管理用户的化妆品，记录寿命并生成专属配方卡。
功能明细：
寿命寿命警告器：设定可用天数（例如 180 天），进度条随着天数减少而缩短。当保质期寿命低于 25% 或 90 天时，进度条变成醒目的警示红，并在“个人美学中心”醒目提醒。
妆容方案绑定（特色功能）：
分类导航第一个为高亮金底“妆容”方案。
用户可创建整套妆容，上传妆容图片，再分别定义并关联：“眼妆用的眼影”、“唇色口红”、“底妆粉底”、“骨相修容”、“高光亮粉”、“鼻影雕刻”等具体单品，提供最严丝合缝的搭配参照。

2.5 个人中心与美学中心 (Profile Page)
功能描述：总结用户的个人美学倾向与衣柜、化妆品全景指标。
特色产出：
最高频率明星单品：透视展示穿着次数最多的那件衣物（例如海盐蓝天丝衬衫），见证陪伴时光。
流转频率监控：监控衣物利用率，防止时尚资产闲置浪费。

3. 非功能性需求与交互美学规范 (UI/UX Best Practices)
3.1 视觉设计准则 (Artistic Flair Theme)
本款产品避开了庸俗的渐变色和粗笨的圆角框，遵循高级画廊的现代极简主义美感：
背景主基调 (Base Color)：采用奶酪杏白（#FAF9F6）取代刺眼的纯白；
主色调 (Sage Green)：深灰鼠尾草绿（#5A5A40），低调内敛；
点缀色 (Gilded Gold)：香槟流沙金（#C5A059），彰显法式优雅气质；
字体设置：标题选用精致感极强的 Cormorant Garamond (罗马古典衬线体)，正文和数据选用 Inter。
3.2 数据处理规范 (Data Lifecycle)
全本地离线存储：采用 React Lazy Hook 自主同步至 localStorage。任意单件的添加、穿着次数变化都会在毫秒级更新并保留在本地，提供不受断网和会话过期影响的用户体验。
